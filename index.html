    • Welcome

    Welcome to the AB Tasty for app, ABTastySdk documentation !

    The ABTastySDK project is an iOS Framework that helps you to make server side tests on your iOS app.

    You can use this FrameWork to access AB Tasty endpoints, which can generate a unique visitor ID, allocate a visitor to a test, and push visits and conversions events in order to help you analyze the outcomes of your campaigns.

    May you have any question about this documentation, feel free to reach out to us.

    App prerequisites

    • Your app must be a native app written in Swift or Objective C code

    • Your app must be compatible with the minimum sdk version iOS 8.0+


    Your app must be compatible with the minimum sdk version iOS 8.0+

    Getting started

    ABtastySDK is available through Pods and manual installation.

    Cocoapods

    Manual Installation

    Add the dependency

    Then add the following library dependency in the build.gradle file of your app module

    com.abtasty:librarybyapi:x.x.x

    Replace x.x.x with the desired version.


    Warning : As the librarybyapi is really light in terms of methods it also depends on other libraries and then in certain circumstances you may need to enable the multidex option if your app dex file reach the limitation of 65k methods. Follow this link : Multidex


    Initialize and start the library

    Call the function initialize and its inner function start from the class ABTasty to init and start the library

    class MyActivity : Activity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        ABTasty.initialize(applicationContext, AB_TASTY_KEY).start()
    }
}

    public class MyActivity extends Activity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        ABTasty.Companion.initialize(getApplicationContext(), AB_TASTY_KEY).start();
    }
}

    Finally you just have to call the function initialize and its inner function start located in the ABTasty class. Put it at the most appropriate moment depending on your application to init and start the library.

    fun initialize(context: Context, token: String) : ABInitializer

    @See: Initalization section for more details

    Replace AB_TASTY_KEY with the key provided by AB Tasty.

    Starting campaigns

    This section of the documentation aims to help you to start a new campaign in your app using the ABTasty android librarybyapi.

    Library Initialization

    The first thing you need to do is initializing the library. To do this, you just have to call the function initialize in the ABTasty class located in the package com.abtasty.librarybyapi.main. For each new user, at the very first initialization, the library is going to ask for a new visitor id to our servers and save it in a database.


    Initialize

    class MyActivity : Activity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)

        ABTasty.initialize(applicationContext, AB_TASTY_KEY)
    }
}

    public class MyActivity extends Activity {
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);

        ABTasty.Companion.initialize(getApplicationContext(), AB_TASTY_KEY).start();
    }
}

    fun initialize(context: Context, token: String) : ABInitializer

    This function is the main library initializer.

    Parameter Type Description
    context Context Application context of your application.
    token String Access key provided by AB Tasty


    This function returns an `ABInitializer` builder object in which you can pass some of the following initial configurations methods. Then call `start` to build and start the library with the given config.


    Initialize with campaign Ids

    class MyActivity : Activity() {

    override fun onCreate(savedInstanceState: Bundle?) {

    withWhenReady(lambda: () -> Unit) : ABInitializer

    As the sdk is asynchronous and run in parallel, this method allows you to set a lambda which is going to be executed when the library is ready.

    Parameter Type Description
    lambda lambda Lambda that is going to be invoked when the library has finished to initialize itself.

    Allocate variations

     ABTasty.getVariationIdForCampaign("xxxxxx", {
        when (it) {
            "aaaaaa" -> System.out.println('Variation A')
            "bbbbbb" -> System.out.println('Variation B')
            ...
        }
    })

        ABTasty.Companion.getVariationIdForCampaign("xxxxxx", new Function1<String, Unit>() {
            @Override
            public Unit invoke(String s) {
                switch (s) {
                    case "aaaaaa" : {
                        System.out.println("Variation A");
                    } break;
                    case "bbbbbb" : {
                        System.out.println("Variation B");
                    } break;
                }
                return null;
            }
        });

    Once the library is initialized you need to allocate a variation id to your campaign. For this, the library contains a method named getVariationIdForCampaign in the class ABTasty located in the package com.abtasty.librarybyapi.main

    fun getVariationIdForCampaign(campaignId: String, lambda: (variationId: String?) -> Unit)

    It takesthe id of your campaign and a lambda as parameters which is going to be invoked when a variation is allocated to this campaign.

    Parameter Type Description
    campaignId String id of the campaign
    lambda lambda : (variationId: String?) -> Unit code to be invoked when a variation is selected for this campaign. The selected variation will be given as a parameter.
    Replace xxxxxx with the id of your campaign and, aaaaaa and bbbbbb with all the possible variations id of your campaign.

    Activate Campaign

    Once you got a variation allocated for a campaign you need to activate it. This way it tells to AB Tasty that the current user got this variation for this campaign so it wont change and always get the same.

    The allocation can change if you call getVariationIdForCampaign several times until you send a ABCampaignActivationTracker to activate it.

    @See ABCampaignActivationTracker in the tracking section

    Sending events

    The library also provides some functions to send events to ABTasty, for exemple to send objectives about a specific campaigns.

    To do this you have to call the method sendTrackingEvent passing the right tracker as a parameter.

    All the possible trackers are listed below.

    Each tracker contains a Builder class that facilitates its build.


    ABTracker

    ABTracker is the generic tracker that contains common data and from which all others trackers inherit.

    Parameter Type Description
    device_type DeviceType Device used by the visitor to view the test. The devices currently supported are: "DESKTOP", "MOBILE", "TABLET", "CAMERA", "TV", "GAMES_CONSOLE", "PAYMENT_TERMINAL", "WRISTWATCH", and "OTHER". required
    timestamp Long timestamp of the event required
    origin String interface or page from where the event has been created required
    geoPoint DoubleArray  Current latitude / longitude of the device for geolocation [-90:90][-180:180] optional
    ip String current ip of the device optional
    deviceResolution IntArray current screen resolution of the device optional (required for ABActionTracking Touch)
    Common data can be automatically set from the tracker passed in the `ABInitializer` returned from the `initialize function`, or from another shared ABTracker instance.
    We strongly recommend to set the `timestamp` parameter everytime you post asynchronous events in order to ensure the data consistency.
    Parameters : device_type, timestamp, deviceResolution can be automatically determined if not set.


    Campaign Activation Tracking

    val tracker = ABCampaignActivationTracker.Builder()
                .setCampaignId("xxxxxx")
                .setVariationId("bbbbbbb")
                .setOrigin("LoginActivity")
                .setDeviceType(DeviceType.MOBILE_PHONE)
                .setTimestamp(System.currentTimeMillis() / 1000)
                .build()
        ABTasty.sendTrackingEvent(tracker)

        ABCampaignActivationTracker activation = new ABCampaignActivationTracker.Builder()
                        .setCampaignId("xxxxxx")
                        .setVariationId("bbbbbb")
                        .setOrigin("LoginActivity")
                        .setDeviceType(DeviceType.MOBILE_PHONE)
                        .setTimestamp(System.currentTimeMillis() / 1000)
                        .build();
                ABTasty.Companion.sendTrackingEvent(activation);

    This tracker activate a campaign and its variation.

    Parameter Type Description
    campaign_id String campaign id to activate required
    variation_id String variation id of the campaign to activate required
    Replace xxxxxx with the id of your campaign and bbbbbb with the selected variations id of your campaign.
